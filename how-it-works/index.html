Synthetic demonstration data. No real patients.

How It Works

One rule engine. One finding. Three surfaces.

What this is

A demo that reads a queue of simulated prescription refill events and gives every prescription a second look before anyone acts on it. The server evaluates each event against federal dispensing rules and screens each patient’s regimen for drug-drug interactions; the three surfaces — patient, pharmacist, and prescriber — each render the same finding in the language their reader needs. Search answers for any drug, not just the family’s: unknown names fall through to the live FDA label database.

System architecture

The rules

Each rule is one function that takes an event and returns a Finding or nothing.

RuleChecksCatches in this demoCitation
R1 Patient age against the label minimum An 11-year-old prescribed zolpidem, which has no pediatric indication DailyMed SPL 8.4
R2 Refill count and total days supply against the schedule cap 90 tablets with 4 refills — 450 days on a Schedule IV window of 180 21 CFR 1306.12, 1306.22
R3 DEA registration presence and check digit One prescription with no registration on file, one that fails the checksum 21 CFR 1301.13, 1306.03

The DEA check digit is arithmetic, not a lookup: the sum of digits one, three, and five, plus twice the sum of digits two, four, and six, must end in digit seven. R2 does dosage math: it parses the sig for doses per day before computing days supply, so 90 tablets taken twice daily is 45 days, not 90.

The drug interaction screen

Alongside the per-prescription rules, the server screens each patient’s full regimen for known drug-drug interactions. The screen runs per person — an interaction is a property of two prescriptions in one body, not of the queue — and each finding carries a severity, the clinical mechanism, a plain-language version for patients, and a suggested action. On the current queue it finds a major serotonin-syndrome pair (sertraline + tramadol) and two moderate pairs. A test asserts the screen never pairs drugs across different patients.

How search decides what to show

Medication knowledge

Every medication answers the questions a patient actually asks: what it is approved to treat, its commonly prescribed off-label uses, its drug class, and how it works in the body — raises dopamine and norepinephrine, boosts GABA at the GABA-A receptor, blocks histamine H1. All of it is searchable, so typing “benzodiazepine” or “dopamine” surfaces the matching drugs.

Data honesty

The interface once carried a green badge reading “SECURE U.S. CLINICAL STREAM.” The application uses synthetic records and makes no security claim, so the badge is gone. A gray line on every page states what the data actually is. The same honesty applies to the two data tiers: curated entries are hand-written for readability; live lookups show the FDA label verbatim, trimmed but never rewritten, with a link to the full source.

Authoritative sources

  • openFDA — live FDA drug labels: indications, interactions, mechanism, pediatric use.
  • RxNorm (NIH/NLM) — drug name normalization, brand to ingredient.
  • DailyMed (NIH/NLM) — full Structured Product Labeling, linked from every lookup card.
  • eCFR Title 21 — federal refill and registration requirements, cited by every rule finding.
